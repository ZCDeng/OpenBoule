Nº 01 — CAPABILITIES

引擎做什么

01

确定性脚手架

Phase 0 秒级生成项目骨架与目录,不调模型、不耗 token。该用代码答的判断,绝不空转 agent。

02

多角色 agent 编排

researcher / strategy / review-panel / editor / designer 按 7+3 阶段 DAG 分工 fan-out 并发、串行放行闸,合议交付。

03

真实联网检索

researcher 接 Aditly web 工具真检索,带来源 URL 落进报告,不靠模型记忆编造,可超训练截止时点。

04

对抗验证三票

source-verifier 对每条断言独立三票裁决,refute 优先。站不住的论据当场出局,不让似是而非过关。

Nº 02 — CREW / SKILLS

7 个角色,各是一份 skill

R1

行业研究员

industry-researcher

接 Aditly web 工具按轴真检索,带来源 URL 落进报告。Phase 2 并发 fan-out。

R2

对抗声称验证

source-verifier · v2.5

对每条断言独立三票裁决,refute 优先。站不住的论据当场出局;Phase 3.5 评审合议同源驱动。

R3

战略顾问

strategy-advisor

把验证过的发现合成报告,Phase 3 出结构化结论与建议;Phase 3.5 评审判返工时据裁决重写。

R4

审稿编辑

editor

Phase 4 串行三筛 + 语言闸门,代码裁决放行,不靠模型自评。

R5

设计师

designer

Phase 5 把报告排版交付,生成可分享的文档与方法论图。

R6

市场扫描员

market-scanner

Phase 6 增益,扫描热点与新增信号,回灌可追加的调研轴。

R7

信息架构师

information-architect

梳理目录与产物结构,为后续阶段提供可追溯的组织骨架。

+N

下一个角色

your-role.md

加一份角色 prompt 就多一种专长。引擎自动接管编排。

可扩展性

新增一个角色 = 一份 roles/your-role.md 提示词 + 一条 dispatch 映射(mapRoleToFile)。不改引擎核心,fan-out 并发、串行放行、成本与事件流自动接上。
